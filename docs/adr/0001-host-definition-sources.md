# Host definition sources

The language server provides Office object model intelligence from `HostDefinition`s supplied by configured `HostApplication`s. Each supported host application ships with a bundled minimal catalog for immediate and portable behavior, centered on high-frequency root object types, uses a per-host user cache under `~/.vba-language-server/host-catalogs/` when available, and refreshes from that application's COM type information asynchronously on Windows when the application is installed and selected. COM discovery uses Office application automation and representative objects, not direct type library parsing, so detailed host method signature help, constants, and enum values are tracked separately. COM-discovered definitions take precedence because they match the user's installed Office version, while bundled definitions remain the fallback so startup and tests do not depend on Office or COM availability. COM refresh failures are logged and do not produce user-facing warnings when fallback catalog data is available. External reference libraries such as DAO and ADO are separate from host application object models and are not enabled by Access host support.
